# RFCs

Design documents for non-trivial, cross-cutting changes to `dotnet-diagnostics-mcp`.
Each RFC is numbered, lives here for the lifetime of the project, and links back to the
tracking issue that spawned it. RFCs are not implementation tickets — when an RFC is
accepted, the implementation work is broken into sub-issues that reference the RFC.
