feat(cli): add --native-aot-map to CPU sampling (#489)

Add a `--native-aot-map <path-to-map.xml>` option to the CLI's threshold-
gated CPU capture path so NativeAOT targets get name-based MethodIdentity
on sampled frames instead of raw addresses.

## What changed

**Core (`GatedCaptureUseCases`)**
- `WatchAndCapture` gains an optional `NativeAotSymbolResolutionOptions?
  nativeAotSymbols = null` parameter (default null — backward compatible).
- The gated `CpuSample` branch now forwards that option to
  `cpuSampler.SampleAsync`, mirroring what the MCP's `collect_sample` tool
  already does.

**MCP (`CollectEventsTool`)**
- Updated the `WatchAndCapture` call site to pass `nativeAotSymbols: null`
  explicitly now that the parameter is positional (no behavior change).

**CLI (`CliOptions`, `CliCommands`, `CliHelp`, `CliCompletionScripts`)**
- New `NativeAotMapFile` (string?) option backed by `--native-aot-map <path>`.
- `TryValidateCollect` enforces:
  - `--native-aot-map` is only valid with `--capture cpu-sample`; rejected
    when `--capture` is absent, or set to a different capture kind.
  - If the supplied path does not exist, a clear error is returned before any
    collection starts.
- `CollectAsync` builds a `NativeAotSymbolResolutionOptions` from the option
  and passes it to `GatedCaptureUseCases.WatchAndCapture`.
- Help text documents the option under the `collect` section, noting it is
  Linux/perf-path only (ETW path on Windows ignores it — pre-existing
  RoutingCpuSampler behavior, not in scope here).
- Shell completions (bash / zsh / pwsh) complete `--native-aot-map` as a
  file path; it is added to the `collect` command's option list.

## Tests

Four new tests in `CliGatedCaptureValidationTests`:
- `Parse_NativeAotMap_PopulatesField` — round-trips the option.
- `TryValidateCollect_NativeAotMap_MissingFile_Fails` — friendly error when
  the file does not exist.
- `TryValidateCollect_NativeAotMap_WrongCaptureKind_Fails` — rejects when
  `--capture dump` is used with `--native-aot-map`.
- `TryValidateCollect_NativeAotMap_WithoutCapture_Fails` — rejects when
  `--native-aot-map` is supplied without any `--capture`.

All 313 CLI tests pass. Build is zero-warning for source projects.
